# AI 总结和解答功能测试

## 测试清单

### 1. 后端测试

#### 1.1 创建文章并验证 AI 总结生成

```bash
# 假设已有用户登录 token
# 创建一篇文章
curl -X POST http://localhost:3000/api/posts \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "title=番茄种植技巧分享" \
  -F "content=番茄是常见的蔬菜作物，种植时需要注意土壤选择、浇水频率和施肥方法。适宜的土壤pH值在6.0-6.8之间，每天需要至少6-8小时的阳光照射。浇水要保持土壤湿润但避免积水，一般每2-3天浇一次水。施肥可以使用有机肥，如腐熟的鸡粪或牛粪，每月施1-2次。" \
  -F "tags=[\"番茄\", \"种植\", \"技巧\"]"

# 查看文章列表，确认 ai_summary 字段是否有值
curl http://localhost:3000/api/posts

# 查看单篇文章详情
curl http://localhost:3000/api/posts/文章ID
```

#### 1.2 创建问题并验证 AI 解答生成

```bash
# 创建一个问题
curl -X POST http://localhost:3000/api/questions \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "title=黄瓜叶片发黄怎么办？" \
  -F "content=我家种的黄瓜最近叶片开始发黄，是从下部叶片开始的，逐渐向上蔓延。浇水正常，没有明显的病虫害迹象。请问可能是什么原因？该如何处理？" \
  -F "tags=[\"黄瓜\", \"病害\"]"

# 查看问题列表，确认 ai_answer 字段是否有值
curl http://localhost:3000/api/questions

# 查看单个问题详情
curl http://localhost:3000/api/questions/问题ID
```

### 2. 前端测试

#### 2.1 文章 AI 总结测试

1. 打开前端应用 http://localhost:5173
2. 登录系统
3. 进入社区 -> 动态 tab
4. 点击任意文章查看详情
5. 验证：
   - 文章内容下方、评论区上方显示"AI 智能总结"区域
   - 有 AI 图标和"AI 生成"标签
   - 总结内容显示在带背景色的框中
   - 如果文章是新建的，等待几秒后刷新页面应能看到总结

#### 2.2 问题 AI 解答测试

1. 在社区 -> 问答 tab 点击任意问题
2. 验证：
   - 问题内容下方、评论区上方显示"AI 智能解答"区域
   - 如果解答已生成，显示完整解答内容
   - 如果解答还在生成中，显示"AI 解答生成中..."提示
   - 刷新页面后应能看到解答内容

#### 2.3 新建文章/问题测试

1. 点击"写文章"按钮
2. 填写标题、内容、标签
3. 发布文章
4. 立即查看文章详情
5. 验证：
   - 初次可能显示"AI 总结生成中..."（如果实现的话）
   - 等待 5-10 秒后刷新页面
   - 应该能看到 AI 生成的总结

6. 点击"提问题"按钮
7. 填写标题、内容、标签
8. 发布问题
9. 立即查看问题详情
10. 验证：
    - 初次可能显示"AI 解答生成中..."
    - 等待 5-10 秒后刷新页面
    - 应该能看到 AI 生成的解答

### 3. 数据库验证

```sql
-- 检查文章的 ai_summary 字段
SELECT post_id, title, ai_summary FROM posts ORDER BY post_id DESC LIMIT 5;

-- 检查问题的 ai_answer 字段
SELECT question_id, title, ai_answer FROM questions ORDER BY question_id DESC LIMIT 5;
```

### 4. 日志验证

查看后端服务器日志，应该看到类似以下的输出：

```
[AI 总结] 开始为文章 XX 生成总结...
[AI 总结] 文章 XX 总结生成成功: "..."

[AI 解答] 开始为问题 XX 生成解答...
[AI 解答] 问题 XX 解答生成成功: "..."
```

## 预期结果

✅ 文章创建后，异步调用 AI 生成总结并保存到 `ai_summary` 字段
✅ 问题创建后，异步调用 AI 生成解答并保存到 `ai_answer` 字段
✅ 前端详情页正确显示 AI 总结（文章）或 AI 解答（问题）
✅ AI 生成区域有明显的视觉标识（图标、标签、背景色）
✅ 如果 AI 生成失败或未完成，有友好的提示信息
✅ 后端日志清晰，能追踪 AI 生成状态

## 注意事项

1. AI 生成是异步的，不会阻塞创建请求的响应
2. 创建后可能需要等待 5-15 秒才能看到 AI 生成结果
3. 如果未配置 LLM_API_KEY，AI 生成会跳过
4. AI 生失败不影响文章/问题的正常创建
